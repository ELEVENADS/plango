# 数据库设计
### plan-task 核心服务
**1. 表说明**：
表名：paln
存储引擎：InnoDB
字符集：utf8mb4
用途：存储用户所有计划条目，包括手动创建和AI生成的

**2. 字段设计**

| 字段名          | 类型           | 约束                                                             | 说明                                            |
|--------------|--------------|----------------------------------------------------------------|-----------------------------------------------|
| id           | BIGINT       | PK, AUTO_INCREMENT                                             | 主键，自增                                         |
| user_id      | BIGINT       | NOT NULL, INDEX                                                | 用户 ID，关联用户服务，用于数据隔离                           |
| title        | VARCHAR(200) | NOT NULL                                                       | 计划标题，限制长度提高性能                                 |
| description  | TEXT         | NULL                                                           | 详细描述，可空                                       |
| plan_date    | DATE         | NOT NULL, INDEX                                                | 计划执行日期，常用查询条件                                 |
| start_time   | TIME         | NULL                                                           | 计划开始时间（可选）                                    |
| end_time     | TIME         | NULL                                                           | 计划结束时间（可选）                                    |
| priority     | TINYINT      | NOT NULL DEFAULT 0                                             | 优先级：0=低，1=中，2=高                               |
| status       | VARCHAR(20)  | NOT NULL DEFAULT 'PENDING', INDEX                              | 状态：PENDING, IN_PROGRESS, COMPLETED, CANCELLED |
| ai_generated | TINYINT(1)   | NOT NULL DEFAULT 0                                             | 是否由 AI 生成：0=否，1=是                             |
| ai_feedback  | TEXT         | NULL                                                           | AI 生成时的额外反馈或建议                                |
| tags         | VARCHAR(500) | NULL                                                           | 标签，用逗号分隔（可后续扩展为关联表）                           |
| created_at   | DATETIME     | NOT NULL DEFAULT CURRENT_TIMESTAMP                             | 创建时间                                          |
| updated_at   | DATETIME     | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间，自动更新                                     |
| deleted      | TINYINT(1)   | NOT NULL DEFAULT 0, INDEX                                      | 逻辑删除：0=正常，1=已删除                               |
| ext_info     | JSON         |                                                                | 扩展信息，存储临时或不确定的结构化数据                           |
**3. 索引设计**
**说明:**
用户重点查询某天计划，所有建立用户和日期的联合索引
```
PRIMARY KEY (`id`),
INDEX `idx_user_id` (`user_id`),
INDEX `idx_plan_date` (`plan_date`),
INDEX `idx_status` (`status`),
INDEX `idx_deleted` (`deleted`),
INDEX `idx_user_date` (`user_id`, `plan_date`)   -- 联合索引，加速用户按日期查询
```
