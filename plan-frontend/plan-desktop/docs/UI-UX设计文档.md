> **关联文档**：[[桌面端规划说明.md|工程规划]]

## 一、设计理念

### 1.1 整体风格：暗色专业风（Dark Professional）

PlanGo Desktop 定位为**高效日程管理工具**，视觉风格以暗色为主基调，融合渐变与微动效，营造**专注、冷静、科技感**的使用氛围。

- **克制**：大面积暗色背景减少视觉噪音，让内容（计划、日程）成为焦点。
- **层次**：通过灰度梯度、透明度叠加、毛玻璃效果构建 Z 轴纵深。
- **呼吸感**：留白充裕，信息密度控制在中低水平，避免工具型产品常见的拥挤感。
- **微动效**：css-doodle 四叶草动画作为品牌视觉锚点，低透明度置于背景层，不抢夺注意力。

### 1.2 情感关键词

| 维度     | 关键词                       |
| -------- | ---------------------------- |
| 氛围     | 专注、沉稳、科技、现代        |
| 节奏     | 从容、有条理、不紧不慢        |
| 品牌感   | 有机（四叶草）、智能（AI）、信赖 |

## 二、视觉基础系统

### 2.1 色彩系统

#### 主色调（Neutral / Surface）

```
#000000    纯黑 — 内容区底色、底部
#0A0A0A    近黑 — 主背景、卡片底色
#1C1C1C    深灰 — 次级底板（Hero 渐变起点）
#333333    中暗灰 — 输入框底色、骨架屏
```

#### 辅助灰度（Border / Disabled / Muted）

```
#BCBCBC    亮灰 — 渐变终点、分隔线
#D9D9D9    浅灰 — 主按钮填充、禁用态文字
rgba(255,255,255,0.6)   半透白 — 次级文字、图标
rgba(255,255,255,0.4)   更透白 — 占位文字、禁用元素
rgba(255,255,255,0.05)  微透白 — 分割线、卡片边框
```

#### 品牌点缀色

```
#23A523    四叶草绿 — 品牌动画主色（css-doodle clover 图形）
```

#### 功能色（Functional）

```
#FF6B6B    错误/删除 — 红色
#51CF66    成功/完成 — 绿色
#339AF0    信息/链接 — 蓝色
#FCC419    警告/提醒 — 黄色
```

#### 计划优先级色标（Priority Tags）

```
P0 紧急    #FF6B6B    红
P1 高      #FF922B    橙
P2 中      #339AF0    蓝
P3 低      #868E96    灰
```

#### 渐变规范

| 用途           | 渐变值                                                       |
| -------------- | ------------------------------------------------------------ |
| 导航栏         | `linear-gradient(90deg, #0A0A0A 65%, rgba(188,188,188,0.74) 100%)` |
| Hero 展示板    | `linear-gradient(90deg, #0A0A0A 65%, #BCBCBC 100%)`          |
| 计划卡片高亮态 | `linear-gradient(135deg, rgba(255,255,255,0.03) 0%, rgba(255,255,255,0.08) 100%)` |

> **原则**：渐变仅用于结构性背景（导航、Hero），不用于正文内容区域，避免视觉疲劳。

### 2.2 字体系统

**字体族**：`Inter`（主字体，覆盖所有文本），降级为 `system-ui, -apple-system, sans-serif`。

| 层级   | 字号   | 字重       | 用途                             |
| ------ | ------ | ---------- | -------------------------------- |
| H0     | 64px   | Regular 400 | 品牌标题（Hero 中的 PlanGoDaily） |
| H1     | 48px   | Regular 400 | 页面标题、LOGO                    |
| H2     | 32px   | SemiBold 600 | 副标题、导航链接                  |
| H3     | 24px   | SemiBold 600 | 区块标题、卡片标题                 |
| Body   | 16px   | Regular 400 | 正文、描述文本                     |
| Small  | 14px   | Regular 400 | 辅助信息、时间戳                   |
| Tiny   | 12px   | Regular 400 | 标签、徽标、快捷键提示             |

**行高**：标题 `1.2`，正文 `1.5`，列表项 `1.4`。  
**字间距**：LOGO 类大字 `letter-spacing: 2px`，其余默认 `0`。

## 三、组件设计规范

### 3.1 按钮（Button）

#### 3.1.1 主按钮（Primary / CTA）

用于页面的核心操作入口（如 Hero 中的 GET START）。

| 属性       | 值                |
| ---------- | ----------------- |
| 尺寸       | 215 × 64px        |
| 圆角       | 24px              |
| 背景       | `#D9D9D9`         |
| 文字颜色   | `#000`             |
| 字号/字重  | 32px / SemiBold 600 |
| 字体       | Inter              |

**状态变化**：
- **默认**：`#D9D9D9` 背景
- **Hover**：背景提亮至 `#E8E8E8`，`transform: scale(1.02)`，过渡 150ms
- **Active**：背景压暗至 `#C0C0C0`，`transform: scale(0.98)`
- **Disabled**：背景 `#555`，文字 `rgba(255,255,255,0.3)`，cursor not-allowed

#### 3.1.2 次级按钮（Secondary）

用于非主要操作（取消、返回、次要入口）。

| 属性       | 值                          |
| ---------- | --------------------------- |
| 尺寸       | 120 × 40px（常规），可伸缩   |
| 圆角       | 12px                        |
| 背景       | 透明                        |
| 边框       | 1px solid `rgba(255,255,255,0.2)` |
| 文字颜色   | `rgba(255,255,255,0.8)`      |
| 字号/字重  | 16px / Medium 500            |

**状态变化**：
- **Hover**：边框变亮 `rgba(255,255,255,0.4)`，背景 `rgba(255,255,255,0.05)`
- **Active**：背景 `rgba(255,255,255,0.1)`

#### 3.1.3 文字按钮 / 链接（Text Button）

用于导航链接、行内操作。

| 属性       | 值                              |
| ---------- | ------------------------------- |
| 字号/字重  | 继承上下文（导航 32px/Regular） |
| 颜色       | `#fff`（导航）/ `#339AF0`（链接） |
| 下划线     | 仅 Hover 时出现                |

**状态变化**：
- **Hover**：opacity 0.7 → 1，下划线渐入（Link 型）

#### 3.1.4 图标按钮（Icon Button）

用于通知铃铛、关闭、更多操作等。

| 属性       | 值              |
| ---------- | --------------- |
| 尺寸       | 40 × 40px       |
| 圆角       | 50%（圆形）     |
| 背景       | 透明            |

**状态**：Hover 时显示圆形 `rgba(255,255,255,0.08)` 背景。

### 3.3 计划卡片（Plan Card）

这是应用中最核心的 UI 单元，出现在 Dashboard、PlanList 中。

| 属性       | 值                            |
| ---------- | ----------------------------- |
| 尺寸       | 宽度自适应（max 640px），高度自适应 |

**卡片内容布局**（从上到下）：
1. **顶部行**：优先级色标（4px 高 × 卡片全宽，置于卡片顶部）+ 时间段文字（Small/14px）

3. **描述**：Body / 16px / Regular，最多 2 行截断
4. **底部行**：标签列表（Tiny 标签）+ 完成状态勾选框（右侧）

**状态**：
- **Hover**：边框 `rgba(255,255,255,0.15)`，`transform: translateY(-2px)`，阴影 level-1
- **已完成**：整体 opacity 0.6，标题添加删除线
- **拖拽中**：阴影 level-2，`transform: scale(1.02)`

## 四、页面布局规划

### 4.1 整体路由与页面结构

```
/login            →  登录页（独立布局，无 Sidebar）
/portal           →  门户页（顶栏 + Hero + 内容 + 底部，无 Sidebar）
/                 →  今日概览 Dashboard（Sidebar + 内容区）
/plans            →  计划列表（Sidebar + 内容区）
/plans/:id        →  计划详情（Sidebar + 内容区）
/plans/create     →  创建计划（Sidebar + 内容区）
/settings         →  个人设置（Sidebar + 内容区）
```

```
│      │  └──────────────────────────────┘   │
└──────┴─────────────────────────────────────┘
```

**编辑模式**：点击"编辑"按钮后，各字段变为可编辑状态（行内编辑），底部出现"保存"和"取消"按钮。

- 预览以卡片形式展示，用户可确认保存或继续调整
- 整个过程展示 Loading 骨架屏

## 五、交互规范

### 5.1 过渡与动画

| 类型       | 时长   | 缓动函数                           | 适用场景                 |
| ---------- | ------ | ---------------------------------- | ------------------------ |
| 微交互     | 150ms  | `ease-out`                         | Hover、Focus 状态切换     |
| 标准过渡   | 250ms  | `cubic-bezier(0.4, 0, 0.2, 1)`    | 页面元素出现/消失、展开   |
| 页面切换   | 300ms  | `cubic-bezier(0.4, 0, 0.2, 1)`    | 路由切换过渡              |
| 弹性动效   | 400ms  | `cubic-bezier(0.175, 0.885, 0.32, 1.275)` | 弹窗打开、FAB 展开 |
| 呼吸循环   | 2000ms | `ease-in-out` + `infinite`         | 四叶草背景动画            |

### 5.2 反馈机制

| 操作类型       | 反馈方式                                             |
| -------------- | ---------------------------------------------------- |
| 保存成功       | Toast（顶部居中，2s 自动消失），绿色对勾图标           |
| 保存失败       | Toast + 错误原因文字，红色叉号图标，4s 自动消失        |
| 删除操作       | 确认弹窗 → 执行 → Toast "已删除"，含撤销按钮（5s）     |
| 加载中         | 骨架屏（Skeleton）替代空白加载态，首屏 ≤ 300ms 不显示  |
| 网络异常       | 页面顶部 Banner 提示 + 重试按钮                        |
| WebSocket 断连 | 通知铃铛旁黄色圆点，悬停显示"连接中断，正在重连..."    |

### 5.3 键盘快捷键

| 快捷键       | 作用               | 页面范围       |
| ------------ | ------------------ | -------------- |
| `Ctrl+N`     | 新建计划           | 全局           |
| `Ctrl+S`     | 保存当前编辑       | 编辑模式       |
| `Ctrl+/`     | 显示快捷键面板     | 全局           |
| `Esc`        | 关闭弹窗/退出编辑  | 全局           |
| `←` `→`     | 日期切换（前一天/后一天） | Dashboard |

## 六、响应式适配

桌面端 Electron 窗口默认 **1200×800**，支持用户拉伸至 **800×600（最小）~ 2560×1440（最大）**。

| 视口宽度     | 适配策略                                     |
| ------------ | -------------------------------------------- |
| ≥ 1440px     | 完整布局：Sidebar 展开 + 计划卡片 2 列        |
| 1200–1439px  | Sidebar 展开 + 计划卡片 1 列（默认）          |
| 960–1199px   | Sidebar 折叠（仅图标） + 计划卡片 1 列        |
| 800–959px    | Sidebar 隐藏（汉堡菜单唤出）+ 计划卡片全宽    |

**门户页**：Hero Mockup 在宽度 < 1080px 时隐藏，仅保留左侧文字内容。

---

## 七、无障碍（A11y）

- **对比度**：正文与背景对比度 ≥ 4.5:1（WCAG AA 标准）
- **焦点指示器**：所有可交互元素有可见的 focus ring（`#339AF0` 2px 外发光）
- **键盘导航**：Tab 顺序合理，弹窗打开时焦点锁定在弹窗内
- **屏幕阅读器**：关键操作按钮提供 `aria-label`，图标提供语义化描述
